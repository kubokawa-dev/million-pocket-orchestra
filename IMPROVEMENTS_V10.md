## 従来モデルの根本的な問題点

### ❌ 問題1: 過去の当選番号を完全除外
```python
# 従来の実装
if num_str not in historical_numbers:  # 過去の当選番号を除外
    predictions.append(num_str)
```
**問題**: 実際には数字の再出現が頻繁に起こる

### ❌ 問題2: 同じ数字の複数出現を軽視
- 656のように同じ数字が複数桁に出現するパターンを予測できない

### ❌ 問題3: 桁の継続性を軽視
- 631 → 656のように、特定の桁が連続するパターンを見逃す

### ❌ 問題4: 大きな変化を軽視
- ±1-2の微小変化のみ考慮、±3-5の大変化を見逃す

**重み**: 30.0（全モデル中最高）


## 技術的詳細

### 1. 数字再出現の検出
```python
# 直近30回で各数字の出現頻度
all_digits = []
for _, row in recent_30.iterrows():
    all_digits.extend([row['d1'], row['d2'], row['d3']])

digit_freq = Counter(all_digits)
top_digits = [d for d, _ in digit_freq.most_common(8)]

# 頻出数字で同じ数字が複数回出現するパターンを生成
for digit in top_digits:
    for third in range(10):
        if third != digit:
            patterns = [
                f"{digit}{digit}{third}",  # 661, 662, ...
                f"{digit}{third}{digit}",  # 616, 626, ...
                f"{third}{digit}{digit}"   # 166, 266, ...
            ]
```

### 2. 桁継続の実装
```python
latest_digits = [df.iloc[-1]['d1'], df.iloc[-1]['d2'], df.iloc[-1]['d3']]

# 1桁目を継続
for d2 in range(10):
    for d3 in range(10):
        num_str = f"{latest_digits[0]}{d2}{d3}"  # 6xx

# 2桁を継続
for d3 in range(10):
    num_str = f"{latest_digits[0]}{latest_digits[1]}{d3}"  # 63x
```

### 3. 過去当選番号の扱い
```python
# v10.0の変更
if num_str != latest_number:  # 前回のみ除外
    predictions.append(num_str)

# 従来（廃止）
if num_str not in historical_numbers:  # 全ての過去当選番号を除外
    predictions.append(num_str)
```
